# Olist-sql-analysis
Olist 电商平台 SQL 数据分析项目


## 1. 项目简介

本项目基于 **Olist Brazilian E-Commerce Public Dataset**，使用 **MySQL 8.0** 对巴西电商平台的订单、用户、商品、卖家、支付、物流和评价数据进行系统分析。

项目从数据导入与质量校验开始，围绕平台经营表现、销售趋势、商品品类结构、物流履约、用户满意度、卖家表现、支付行为和 RFM 用户分层等方向展开分析，最终识别平台的核心业务特征、潜在问题，并提出相应的运营优化建议。

本项目不是简单的 SQL 查询练习，而是按照真实电商数据分析项目的思路完成：

- 数据库建表与 CSV 数据导入
- 数据完整性与字段格式校验
- 电商核心指标计算
- 多维度业务分析
- 用户价值分层
- 业务问题诊断与优化建议输出

## 5. 分析口径说明

| 指标 / 口径 | 说明 |
|---|---|
| GMV | `SUM(price + freight_value)`，即商品价格与运费之和 |
| AOV | `GMV / 订单数`，表示平均每单成交金额 |
| 有商品明细的订单数 | `orders` 与 `order_items` 内连接后的订单数，用于 GMV、AOV 等交易金额指标 |
| 有商品明细订单覆盖用户数 | 上述订单覆盖到的 `customer_unique_id` 去重用户数，为 95,420 |
| 平台真实去重用户数 | `customers` 表按 `customer_unique_id` 去重，为 96,096 |
| 取消率 | 按订单级 `order_id` 去重计算，不按商品明细行计算 |
| 成交类分析 | 主要排除 `canceled` 和 `unavailable` 订单 |
| 销售趋势分析 | 仅统计 2017-01 至 2018-08，排除 2016 年不完整数据和 2018 年 9 月后异常数据 |
| 物流分析 | 仅统计 `delivered` 且实际送达时间、预计送达时间非空的订单；超时判断采用日期级口径：`DATEDIFF(实际送达日期, 预计送达日期) > 0` |
| 品类缺失 | 部分商品品类为空，SQL 中统一清洗为 `unknown` |
| 评价联表分析 | 涉及 `order_reviews` 的评分分析先按订单级聚合评分后再联表，评分相关口径可能与纯交易 GMV 表略有差异 |
| 支付方式占比 | 同一订单可能存在多条支付记录或混合支付，因此支付方式占比相加可能超过 100% |
| RFM 基准日期 | 以 `2018-09-01` 作为用户最近购买时间计算基准 |

- 2017 年 11 月出现明显销售高峰
- 2018 年订单量与 GMV 进入相对平稳阶段
- GMV 增长主要由订单量扩张驱动，而不是由 AOV 明显提升驱动

| 2017-02 | 1,718 | 283,561.69 | 165.05 | 118.30% | 107.06% |

### 7.5 配送时长与超时分析

在 `04_delivery_analysis.sql` 中，分析整体配送效率、各州配送差异，以及配送超时与用户评分之间的关系。

整体物流表现：

| 指标 | 数值 |
|---|---:|
| 平均实际配送天数 | 12.5 天 |
| 平均承诺配送天数 | 24.4 天 |
| 平均提前送达天数 | 11.9 天 |
| 已送达订单数 | 96,478 |
| 延迟订单数 | 7,826 |
| 整体延迟率 | 8.11% |

分析结论：

平台平均实际配送时间为 12.5 天，而平均承诺配送时间为 24.4 天，实际配送通常早于承诺时间。这说明平台承诺配送日期较为保守，有助于降低用户预期落差。

但仍有 8.11% 的订单发生延迟，且不同州之间存在明显差异。AL、MA、PI、CE、SE、BA 等东北部地区延迟率较高。RJ 州订单量较大，同时延迟率达到 13.47%，应作为重点优化区域。

### 7.6 配送延迟与用户评分关系

以下结果采用日期级超时口径，并且仅统计能关联到 `order_reviews` 的已送达订单：

| 延迟程度 | 订单数 | 平均评分 | 低分率 |
|---|---:|---:|---:|
| 准时 | 89,943 | 4.29 | 9.28% |
| 超时 1-3 天 | 1,856 | 3.29 | 32.17% |
| 超时 4-7 天 | 1,756 | 2.10 | 67.65% |
| 超时 8-14 天 | 1,452 | 1.68 | 80.03% |
| 超时 14 天以上 | 1,353 | 1.74 | 77.90% |

分析结论：

配送延迟与低评分存在明显相关关系。超时 1-3 天时，平均评分为 3.29；超时 4-7 天时，平均评分进一步降至 2.10，低分率达到 67.65%。因此，控制配送延迟可作为改善用户体验的重点运营方向。

### 7.9 RFM 用户分层分析

在 `06_rfm_user_segmentation.sql` 中，基于 RFM 模型对用户进行价值分层。

RFM 评分规则：

| 维度 | 3 分 | 2 分 | 1 分 |
|---|---|---|---|
| R 最近购买 | 180 天以内 | 180-360 天 | 360 天以上 |
| F 购买频次 | 3 次及以上 | 2 次 | 1 次 |
| M 消费金额 | 500 雷亚尔及以上 | 200-500 雷亚尔 | 200 雷亚尔以下 |

用户分层结果：

| 用户类型 | 人数 | 占比 | 特征 |
|---|---:|---:|---|
| 普通用户 | 63,662 | 67.02% | 未表现出明显高消费、高复购或长期流失特征 |
| 流失用户 | 21,749 | 22.90% | 长期未购买且购买频次低 |
| 新客高消费 | 7,649 | 8.05% | 近期首购且消费金额较高 |
| 高价值用户 | 745 | 0.78% | 近期活跃、复购且消费金额较高 |
| 潜力用户 | 618 | 0.65% | 有复购和一定消费能力，但需要继续维护 |
| 沉睡老客 | 560 | 0.59% | 曾多次购买但近期活跃度低 |

分析结论：

普通用户占比最高，为 67.02%；流失用户占 22.90%，说明平台存在较明显的留存问题。高价值用户仅 745 人，占 0.78%，规模较小但具有较高运营价值。新客高消费用户占 8.05%，这类用户具备从首购转向复购的潜力，是后续运营激活的重点对象。

## 9. 业务建议

基于以上分析结果，提出以下优化建议：

1. **优先优化高延迟地区物流**

   AL、MA、PI、CE、SE、BA、RJ 等地区延迟率较高，建议通过区域仓、本地物流合作、配送路线优化或承诺时效重估降低超时风险。

2. **建立高 GMV 低评分卖家监控机制**

   对销售额高但评分偏低的卖家进行履约、售后和商品质量跟踪，避免头部卖家带来集中差评，影响平台整体口碑。

3. **维护核心健康品类**

   `health_beauty`、`watches_gifts`、`sports_leisure` 等高 GMV 高评分品类应保持供给稳定，并强化库存和物流保障。

4. **扶持高客单价小众品类**

   `computers`、`home_appliances_2` 等品类 AOV 高但订单量低，可结合分期支付、保修服务、精准推荐和详情页优化作为转化优化方向。

5. **提升新客复购转化**

   对新客高消费用户在首购后 30 天内进行优惠券、会员权益或相关品类推荐，提升二次购买概率。

6. **优化分期支付展示**

   高金额订单更倾向于使用长期分期，建议在高客单价品类详情页突出分期金额、免息信息和售后保障。

- 分析对象覆盖订单、用户、商品、卖家、支付、物流、评价等多个电商核心模块

- 构建 GMV、AOV、取消率、延迟率、平均评分、复购、RFM 等核心业务指标
- 将 SQL 查询结果进一步转化为业务结论和运营建议
- 通过 RFM 模型完成用户分层

## 12. 项目总结

本项目基于 Olist 巴西电商公开数据集，使用 MySQL 完成了一个较完整的电商 SQL 数据分析项目。

项目从数据导入和质量校验开始，围绕平台经营表现、销售趋势、商品品类、物流配送、卖家支付和用户分层等方向展开分析。分析结果显示，平台整体交易规模较大，订单取消率较低，用户评分整体较好；但同时也存在用户复购偏弱、部分地区物流延迟明显、部分高销售品类评分不足、头部卖家服务质量分化、高价值用户占比较低等问题。

该项目能够体现 SQL 数据处理能力、电商业务理解能力、指标体系搭建能力、用户分层分析能力和数据分析报告输出能力。
